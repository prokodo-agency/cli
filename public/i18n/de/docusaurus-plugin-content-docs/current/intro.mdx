# prokodo CLI

Die **prokodo CLI** ist die offizielle Befehlszeilenschnittstelle für den [prokodo Marketplace](https://www.n8n-marketplace.prokodo.com). Sie ermöglicht es, sich zu authentifizieren, Projekte zu konfigurieren, Dateien zur Cloud-Verifizierung hochzuladen und die Umgebung zu inspizieren — alles ohne das Terminal zu verlassen.

## Was kannst du damit machen?

| Befehl                        | Zweck                                                    |
| ----------------------------- | -------------------------------------------------------- |
| [`auth`](commands/auth)       | API-Schlüssel speichern, rotieren und prüfen             |
| [`init`](commands/init)       | `.prokodo/config.json` im Projekt anlegen                |
| [`verify`](commands/verify)   | Projektdateien hochladen und Cloud-Verifizierung starten |
| [`credits`](commands/credits) | Guthaben abfragen                                        |
| [`doctor`](commands/doctor)   | Umgebung mit einem Befehl diagnostizieren                |

## Designprinzipien

- **CI-first** — jeder Befehl unterstützt `--json`-Ausgabe und die Umgebungsvariable `PROKODO_API_KEY`
- **Vorhersehbare Exit-Codes** — `0` Erfolg · `1` bekannter Fehler · `2` Nutzungs- / Konfigurationsfehler
- **Keine Überraschungen** — nicht-interaktiver Modus wird automatisch erkannt; Eingabeaufforderungen nur wenn stdin ein TTY ist
- **Einzelne Binary** — mit [tsup](https://tsup.egoist.dev) gebaut, alle Abhängigkeiten gebündelt, Node ≥ 22

## Schnellstart

```bash
# 1. Installieren
npm install -g @prokodo/cli

# 2. Authentifizieren
prokodo auth login --key pk_live_…

# 3. Projekt konfigurieren
prokodo init --slug mein-projekt

# 4. Verifizieren
prokodo verify
```

<Admonition type="tip">
  Führe <code>prokodo doctor</code> jederzeit aus, um einen vollständigen Gesundheitscheck der
  Umgebung, Zugangsdaten und API-Verbindung zu erhalten.
</Admonition>
