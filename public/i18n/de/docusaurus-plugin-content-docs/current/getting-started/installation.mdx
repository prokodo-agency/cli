# Installation

## Voraussetzungen

| Voraussetzung     | Version          |
| ----------------- | ---------------- |
| Node.js           | ≥ 22             |
| npm / pnpm / yarn | aktuelle Version |

:::tip
Nicht sicher, welche Version installiert ist? Führe `node --version` aus. Falls Node 22 benötigt wird, verwende [nvm](https://github.com/nvm-sh/nvm): `nvm install 22 && nvm use 22`.
:::

## Global installieren

```bash
npm install -g @prokodo/cli
```

```bash
# oder mit pnpm
pnpm add -g @prokodo/cli
```

```bash
# oder mit yarn
yarn global add @prokodo/cli
```

Installation überprüfen:

```bash
prokodo --version
```

Die aktuelle Versionsnummer sollte auf stdout ausgegeben werden.

## Ohne globale Installation (npx)

Die CLI lässt sich auch ohne globale Installation ausführen:

```bash
npx @prokodo/cli --help
```

## Umgebung prüfen

Nach der Installation den eingebauten Gesundheitscheck ausführen:

```bash
prokodo doctor
```

Erwartete Ausgabe (grüne Häkchen für alle fünf Prüfungen):

```
✓  Node version               22.x.x (erforderlich ≥ 22)
✗  API key configured         Quelle: keine
✗  .prokodo/config.json       Keine Konfigurationsdatei im aktuellen Verzeichnis
…
```

Die Prüfungen `API key configured` und `.prokodo/config.json` schlagen fehl, bis die [Authentifizierung](./quick-start) abgeschlossen ist — das ist erwartet.

## Aktualisieren

```bash
npm update -g @prokodo/cli
```

## Deinstallieren

```bash
npm uninstall -g @prokodo/cli
```
